‚ö° The Tech Stack

React + Supabase in a single HTML file. No build tools, no complexity. Just what we needed to make chores less painful.

Why a single HTML file?

React loaded via CDN. No build step, no webpack, no npm install. Edit the file, refresh the browser, see the changes. Perfect for rapid iteration when you're building with your kids on a weekend.

Why Supabase?

PostgreSQL database with a free tier. No server code to write. The kids could see their points update in real-time. Plus, it forced me to learn Row Level Security (which was valuable).

What the Kids Designed

Point system: They decided which chores were worth more points based on how much they hated doing them. "Vacuum Living Room" got 500 points. "Get mail" got 250.
Cooldown timers: After watching someone claim "Feed Pets" three times in one day, they invented cooldowns. Daily chores = 24hr cooldown. Weekly chores = 7 days.
GOAT competition: The "Greatest Of All Time" award for weekly winner. They wanted bragging rights. They got bragging rights.
Approval workflow: Parents approve chores before points count. This was non-negotiable after someone claimed "Clean Room" without actually cleaning their room.

Recent Bug Fixes

Bug: "View Profile" button was overlapping on mobile. Fix: Added box-sizing constraints and responsive padding.
Bug: Chore names too long on iPhone. Fix: Added text truncation with ellipsis, 2-line clamp.
Bug: Recent Completions showing "Unknown: Unknown" after admin updates. Fix: Added loadAllCompletions() calls to reload data with proper joins.
Feature request: "We can't see the buttons from across the room." Fix: Increased all font sizes for tablet view.

üí° Key Lessons Learned

Build with your users

The kids designed the features. I wrote the code. We tested it together. This is how you build something people actually want to use.

Start simple, add what you need

We didn't build everything on day one. Features emerged from actual use. Cooldowns came from watching someone game the system. Achievements came from wanting more progression.

Complexity is optional

One HTML file. No build tools. No npm install. Just edit, refresh, see changes. Perfect for a weekend project.

Gamification works

Points, leaderboards, achievements, visible progress. The kids argue about who gets to do chores first. That's success.

Bug reports are a gift

When your users find bugs and report them clearly, you're building the right thing for the right people.
